
## Code Study（09:49 UTC+8）

深度阅读了两个核心模块：

### nursing 服务
- 5个聚合根：NursingAdmissionRecord、NursingCarePlan、NursingDiagnosis、MedicationAdministration、TransfusionVerification
- NursingCarePlan 状态机最复杂（draft→active→suspended/resumed/completed/cancelled）
- 发现：状态转换逻辑分散，可引入 FSM 模式统一管理
- 创建 Issue #599

### imaging 服务
- ImagingExecution 聚合根，状态机：REGISTERED→IN_PROGRESS→COMPLETED（或CANCELLED）
- 危急值检测内嵌在 AddObservation 中，领域逻辑不泄漏
- 发现：ImagingObservation.EvaluateFlag() 实现不完整（dead code）；ImagingCriticalValue.direction 用 string 而非枚举
