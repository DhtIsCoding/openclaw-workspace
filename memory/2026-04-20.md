## 🎯 Playwright 网络等待策略经验（14:29 UTC+8）

### 核心经验
**金融行情类页面（东方财富、同花顺等）几乎全部使用 WebSocket 长连接推送实时数据，务必用 `domcontentloaded` 而非 `networkidle`。**

### 判断标准
| 页面特征 | 推荐等待策略 |
|---------|------------|
| 普通内容页（新闻、博客） | `networkidle` ✅ |
| 轮询类实时页面（新浪财经） | `networkidle` 可用 |
| **WebSocket 实时页面（行情、聊天、监控）** | `domcontentloaded` + `waitForTimeout` |
| 需要等特定元素出现 | `waitForSelector` + `domcontentloaded` |

### 错误症状
- `networkidle` 超时 → 几乎可以确认是 WebSocket/长轮询页面
- 页面标题/主框架秒开，但价格数据在几秒后才出现 → 正常，JS 渲染的

### 代码模板
```javascript
// ✅ 推荐：金融行情页
await page.goto(url, { waitUntil: 'domcontentloaded', timeout: 20000 });
await page.waitForTimeout(3000);

// ✅ 推荐：需要等待特定元素
await page.goto(url, { waitUntil: 'domcontentloaded', timeout: 20000 });
await page.waitForSelector('.price', { timeout: 8000 });

// ❌ 禁用：行情/监控类页面
await page.goto(url, { waitUntil: 'networkidle', timeout: 30000 });
```

### 延伸
- 诊断方法：`curl` 能获取 HTML 但 Playwright 超时 → 多半是 WebSocket 或大量外部 iframe
- 所有国内券商行情页、同花顺、东方财富股吧等都适用此经验

## Sub-Agent 最佳实践（15:36 UTC+8）

### 核心原则
**Sub-agent 是异步的，`sessions_spawn` 调用后立即返回，不等待结果。** 结果通过 push 机制（completion event）回到父会话。

### ✅ 推荐写法
```javascript
// ✅ 正确：spawn 后立即返回，等 completion event
const { childSessionKey } = await sessions_spawn({
  task: "具体任务描述",
  runtime: "subagent",
  mode: "run",
  runTimeoutSeconds: 3600, // 必须设超时（常规任务 1 小时）
});
// 不要在这里调用 sessions_list / sessions_history 轮询

// ✅ 超时设置建议
runTimeoutSeconds: 3600  // 常规任务 1 小时（默认）
runTimeoutSeconds: 7200  // 复杂任务 2 小时
runTimeoutSeconds: 0     // 无超时（仅限需要手动控制的长时间任务）
```

### ❌ 常见错误
```javascript
// ❌ 错误1：spawn 后立刻轮询 sessions_history
sessions_spawn({ ... });
await sleep(5000);
sessions_history(childSessionKey); // 不要这样做！

// ❌ 错误2：不设超时
sessions_spawn({ task: "..." }); // 无超时 → 可能产生僵尸任务
```

### 任务生命周期
```
queued → running → succeeded / failed / timed_out / cancelled / lost
```
- `timed_out` = 超时
- `lost` = 子会话 5 分钟内无响应

### Completion 接收方式
- **自动推送**：完成后 result 作为 user message 到达父会话（`sourceTool=subagent_announce`）
- 无需轮询，`sourceSession` 元数据标明了来源
- 父会话收到后转成正常助手语气发给用户

### 父会话通知流程（关键）
```
子agent完成 → sourceTool=subagent_announce 到达父会话
→ 助手收到 completion event
→ 转换为正常语气回复用户
→ 回复末尾不要加 "NO_REPLY" 标记
```

### 子 agent 日志查看
```bash
# 查看子 agent 的会话历史
sessions_history(childSessionKey)

// 查看子 agent 的任务状态
openclaw tasks show <runId>
```

### Session Key 格式
```
agent:main:subagent:<uuid>   # 主agent创建的子agent
agent:<agentId>:subagent:<uuid>  # 其他agent创建
```

### 实用参数组合
```javascript
// 常规任务（推荐默认）
sessions_spawn({
  task: "...",
  runtime: "subagent",
  mode: "run",
  runTimeoutSeconds: 3600,  // 1 小时
});

// 带工具访问
sessions_spawn({
  task: "...",
  runtime: "subagent",
  mode: "run",
  runTimeoutSeconds: 7200,  // 2 小时，复杂任务
  toolsAllow: ["read", "exec", "sessions_history"],  // 可选：限制工具
});

// 持久会话（多轮对话）
sessions_spawn({
  task: "...",
  runtime: "subagent",
  mode: "session",
  runTimeoutSeconds: 0,  // 无超时
});
```

### 监控与调试
```bash
# 查看所有后台任务
openclaw tasks list

# 查看具体任务详情
openclaw tasks show <runId>

# 取消僵尸任务
openclaw tasks cancel <runId>

# 健康检查
openclaw tasks audit
```

### 数据分层采集架构（最终决策）

**L1: 每日积累（自动，~20s/天）**
- 日频价格：收盘/涨跌/量/额/换手 + 板块分类
- K线数据：1d/1h/15m/5m/1m 近10日（滚动覆盖）
- 资金流向：北向/融资融券/龙虎榜
- 市场结构：大宗交易/成分调整

**L2: 每周采集（按需，~30min）**
- 财务报表：营收/净利润/资产负债（季报/年报）
- 估值指标：PE/PB/PS（个股 + 板块均值）
- 股息回购：分红/回购计划
- 限售解禁：未来3个月解禁日程

**L3: 事件驱动（不固定时间）**
- 分析师预期（东财）
- 股东结构变化（季报）
- ETF持仓变动（基金季报）

### SQLite 数据库（轻量方案）
- 路径：`data/daily_close.db`
- 表：`daily_close(symbol, trade_date, close, change_pct)`

## 微信读书《日本蜡烛图技术》抓取（15:43-17:47 UTC+8）

### 背景
dht 想抓取他在微信读书购买的《日本蜡烛图技术》（史蒂夫·尼森）用于个人知识库。

### 技术路线
- **CDP (Chrome DevTools Protocol)** 连接微信读书的 Chrome WebSocket 调试端口（50840）
- **截图 + OCR** 而非直接抓 DOM（书籍正文是 DRM 加密的 JS 动态内容，DOM 中不可见）
- Tesseract OCR + chi_sim+eng 语言包提取图片文字

### 关键发现
1. **微信读书内容无法直接抓取**：正文不在 DOM 中，只有导航框架（68字），书籍内容完全 JS 动态加载且 DRM 加密
2. **正确方案是截图 + OCR**：每页截图（PNG），Tesseract OCR 提取文字
3. **翻页按钮正确选择器**：`.renderTarget_pager_button_right`（BUTTON 元素，不是父级 div）

### 翻页按钮定位过程
- 初期选择器 `[class*=next]` / `[class*=pageNext]` 命中 `readerTopBar_right`（顶部工具栏），导致 25 页全部同一页
- 修复后用 `innerText === "下一页"` 精确定位，发现真正的下一页按钮
- 最终选择器：`.renderTarget_pager_button_right`（第 14 页开始翻页成功）
